# Dreaming v2 — Codex-faithful memory-synthesis pipeline

## Goal
A working, manually-triggered 2-phase memory-synthesis pipeline that replicates
Codex's `codex-memories`. Demonstrable end state: running `/dream` (or
`dream_run.py`) against a set of past Claude session transcripts (a) extracts a
high-signal `raw_memory` per eligible transcript into a `sqlite` store — and
produces NOTHING for a low-signal transcript (the no-op gate) — then (b)
consolidates the selected rows into a self-contained memory workspace, where a
locked-down agent writes `MEMORY.md` + `memory_summary.md` (first line exactly
`v1`), with a second run that adds a transcript surgically updating those files
and a removed transcript surgically forgetting its unsupported content (git-diff
forgetting). The locked-down agent provably cannot write outside the workspace
(post-hoc scope check reverts it). The full spec + the Codex→harness mapping is
`docs/design-docs/dreaming-v2.md`.

## Surprises & discoveries
- 2026-06-13 (M2): verified empirically that Claude Code's project-dir slug is
  every non-alphanumeric char → `-` (incl. `.` and space): `/.claude-worktrees/`
  → `--claude-worktrees-`. So discovery is scoped to THIS repo's dir (matches our
  per-repo workspace) with no content read.
- 2026-06-13 (M2): a transcript's FIRST jsonl record can be a `queue-operation`
  with no `cwd`/message (not a user/assistant turn). M3's content filter must not
  assume the first line carries content or cwd — iterate, don't index [0].
- 2026-06-13 (M2): the deterministic gate caught my own docstring — an
  illustrative `/Users/...` example in `harness_lib.py` tripped S3 (abs-path
  token). harness_lib is exempt from S2 path-*tokens* but not S3 abs-paths.
  Lesson reaffirmed: examples are code too.
- 2026-06-13 (M4): git's rename detection (likely on in global config) collapsed
  a rollout-summary delete+add (structurally similar files) into one `R` change,
  hiding the Delete. Codex's diff is strictly path-based — a forgetting cue MUST
  remain a real Delete — so we pass `--no-renames`. A test caught it; without it
  the M5 agent would never see "this memory was forgotten."
- 2026-06-13 (M5): VERIFIED empirically — a headless `claude -p --allowedTools
  "Write"` writes OUTSIDE its cwd (wrote `/tmp/...`). Claude Code has no
  `writable_roots` enforcement, so the post-hoc scope check is the LOAD-BEARING
  defense, not belt-and-suspenders. Hence the layered posture: tool-min (no
  Bash/network → only Write/Edit can escape) + byte-snapshot revert of every
  git-visible host file + hard-fail/rollback on any escape. (Residual: writes
  OUTSIDE the host repo aren't git-visible → not reverted; mitigated by no-network
  + least-priv + the prompt guard. To revisit in M7/SECURITY.)

## Outcomes & retrospective
**Shipped.** A working, manually-triggered 2-phase memory-synthesis pipeline faithful
to Codex's `codex-memories`: discover eligible transcripts → a **no-tools Haiku** extracts
a high-signal `raw_memory` per rollout into a sqlite store (a no-op gate drops low-signal
ones) → a **locked-down Sonnet** consolidates the selected rows in a workspace it
provably cannot escape (a post-hoc, symlink-safe scope check reverts every out-of-workspace
write byte-for-byte). Wired into `dream_run` (single-flight) behind the `dream-rollouts`
skill. The pipeline was then folded into the memory-as-docs pivot: on a self-hosting repo
Phase 2 ROUTES into the docs tree (read-only router agent + deterministic append-only
applicator) and a docs-sync **forgetting** pass retracts dropped sessions' contributions;
the locked-down flat-store path remains the bare-host fallback.

**What went well.** Write-path-first sequencing meant the load-bearing T2 defense
(`enforce_workspace_scope`) was built and adversarially proven before it carried weight.
The empirical discovery that a headless `claude -p --allowedTools Write` escapes its cwd
(Claude Code has no `writable_roots`) reframed the post-hoc check from belt-and-suspenders
into THE defense — and the M8 gate confirmed it holds against parent-dir symlink swaps at
every depth/ordering. Both accepted reliability P2s proved genuinely bounded under scrutiny.

**What was hard / surprising.** Git's rename detection silently collapsed a forgetting
Delete (fixed with `--no-renames`; a test caught it). The symlink defense took three
review rounds to get right — git-visibility→filesystem-walk boundary, then symlink-following
snapshot, then parent-dir swap → two-pass — each round a real escape that no test caught
until it was written. The M8 final gate ran on Opus subagents rather than Codex because a
concurrent Codex director session had been clobbering the shared worktree (committing
destructive `seed` commits as the user's identity); the work was decoupled into a
standalone clone (`agent-harness-dreaming-v2-solo`) before closing.

**Follow-ons (tracker):** harden `redact_secrets` (vendor-prefix + underscore-glued keys);
wire or formally defer `prune_stage1_outputs`; the Major memory-as-docs portable-propagation.
