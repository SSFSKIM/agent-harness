# Product specs — index

Product specs own the *design* — what to build and why (the `product-design`
skill authors them; an ExecPlan then owns the build). A parent (capability) spec
indexes its children; each child carries a `phase: <initiative>/<NN>-<slug>`
frontmatter key so the roadmap is a **derived** group-by view (`nav.py roadmap`),
never hand-maintained here.

Register each spec below with a one-line description plus its phase/parent (lint
D8). Keep this a catalog, not a status board — live status is the derived
roadmap.
