# docs/.harnessignore — strict-mode legacy doc roots skipped by content lints.
#
# Docs-relative path prefixes, one per line. Matching is on path-segment
# boundaries, so a trailing `/` is optional and forgiving: `business` and
# `business/` both mean the `business/` tree and neither touches a sibling
# `business-plan.md`. A bare filename matches one file (e.g. `README.md`).
# `#` comments and blank lines are ignored.
#
# This is a MIGRATION BACKLOG for hosts that opt into global docs governance
# (`.harness.json` `doc_governance: strict`). Host-owned project docs are
# flexible by default, so most hosts leave this empty. In strict mode, every
# line is a subtree the harness does not yet govern. Migrate a wave
# (frontmatter + kebab-case + index, or `git mv` into a managed tree), then
# delete its line.
#
# Cannot exempt harness-managed trees (design-docs/ memory/ exec-plans/
# product-specs/ generated/) or top-level machine docs
# (PLANS/DESIGN/SECURITY/RELIABILITY/QUALITY_SCORE/PRODUCT_SENSE.md) — those
# entries are ignored; the harness always governs its own tree.
#
# Fresh host: leave this empty. Ported host: only list pre-existing docs/
# subtrees here after choosing strict docs governance.
